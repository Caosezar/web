Fase 1: Componentes Base

Em Planejamento

1.1 Criar Componentes Base Reutilizáveis

Alta

Objetivo: Estabelecer uma biblioteca de componentes reutilizáveis que siga o design system do projeto.

Componentes a criar:

  • Button Component (com variações de tamanho, cor e estado)
  • Input/Form Fields (text, email, password, textarea)
  • Card Component (com header, body, footer)
  • Alert Component (success, error, warning, info)
  • Badge Component (labels e status indicators)
  • Loading Skeleton (para estados de carregamento)
  • Empty State Component (para listas vazias)

Tecnologia: Angular Standalone Components, Material Design

Escopo: Implementar com dark mode, responsivo, acessibilidade (WCAG)

1.2 Dialog Component

Alta

Objetivo: Criar componente de diálogo modal reutilizável para confirmações, formulários e notificações.

Funcionalidades:

  • Dialog genérico com header, body, footer customizáveis
  • Service para gerenciar múltiplos diálogos
  • Confirmação com botões (OK, Cancelar, Customizados)
  • Animations (fade in/out)
  • Teclado: ESC para fechar, TAB para navegar
  • Backdrop (clique fora para fechar)
  • Dark mode support

Baseado em: MatDialog do Angular Material

1.3 Tabela Paginada

Alta

Objetivo: Componente de tabela reutilizável com paginação, sorting e filtros.

Funcionalidades:

  • Renderização de dados com colunas customizáveis
  • Paginação (tamanho da página, navegação)
  • Sorting (ascendente/descendente)
  • Filtros por coluna
  • Seleção de linhas (single e multiple)
  • Ações por linha (editar, deletar, custom)
  • Estados de carregamento e vazio
  • Responsivo (mobile com accordion)

Baseado em: MatTable do Angular Material

Fase 2: Sistema de Autenticação

Em Planejamento

2.1 Módulo de Login

Crítica

Objetivo: Implementar tela e lógica de login com validações e feedback de erro.

Funcionalidades:

  • Formulário reativo (email/CPF + password)
  • Validações em tempo real
  • Feedback de erro detalhado
  • Botão "Esqueci minha senha" (placeholder para fase posterior)
  • Loading state durante autenticação
  • Integração com AuthService (mock → real)
  • Redirecionamento automático se já autenticado
  • Dark mode support

Rota: /login (PublicLayout)

2.2 Fluxo de Autenticação

Crítica

Objetivo: Implementar fluxo completo de autenticação com persistência e proteção de rotas.

Funcionalidades:

  • Auth Guard para proteger rotas /app/*
  • HTTP Interceptor para adicionar token em requisições
  • Persistência de sessão (localStorage)
  • Refresh de token automático (se existir backend)
  • Logout com limpeza de estado
  • Redirecionamento automático para login quando sessão expirar
  • Estados de carregamento inicial (splash screen)

Arquivos:

  • guards/auth.guard.ts
  • interceptors/auth.interceptor.ts
  • services/auth.service.ts (expandir)

2.3 Sign Up / Registro

Alta

Objetivo: Criar fluxo de cadastro de novos usuários.

Funcionalidades:

  • Formulário com validação de força de senha
  • Confirmação de email (verificação)
  • Termos de serviço (checkbox)
  • Feedback de sucesso/erro
  • Link para login se já tem conta

Rota: /signup (PublicLayout)

2.4 Recuperação de Senha

Média

Objetivo: Implementar fluxo de reset de senha.

Funcionalidades:

  • Página de esqueci senha (email/CPF)
  • Envio de link de reset (mock)
  • Página de reset com nova senha
  • Validação de segurança

Rotas: /forgot-password, /reset-password/:token

Fase 3: Funcionalidades Principais (Próximas)

Planejado

3.1 Gerenciamento de Leads

Alta

Objetivo: CRUD completo para gerenciar leads.

Funcionalidades:

  • Listagem com tabela paginada
  • Busca e filtros avançados

3.2 Configurações de Usuário

Média

Objetivo: Página de settings do usuário.

Funcionalidades:

  • Editar perfil (nome, email, foto)
  • Alterar senha
  • Preferências (tema, idioma, notificações)
  • Segurança (2FA, sessões ativas)

Rota: /app/settings

Notas Importantes

  • Dark Mode: Todos os componentes devem suportar dark mode via CSS variables
  • Responsividade: Mobile-first, testar em múltiplas resoluções
  • Acessibilidade: WCAG 2.1 AA, keyboard navigation, screen readers
  • Testes: Unit tests (Jasmine) e E2E (Cypress) para funcionalidades críticas
  • Performance: Lazy loading de módulos, OnPush change detection
  • Backend: Atualmente mock, integração com API real será feita incrementalmente
